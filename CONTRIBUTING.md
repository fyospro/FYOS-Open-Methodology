# Contributing

Thank you for your interest in improving FYOS methodology documentation.

## What We Welcome

### Methodology Questions

If something is unclear or could be explained better:
- Open an issue describing the confusion
- Suggest improved wording or examples
- Ask for clarification on specific concepts

### Error Corrections

If you spot a factual error:
- Open an issue with the specific error
- Include what you believe the correct information is
- Cite sources if applicable

### Additional Examples

If you have examples that would help illustrate concepts:
- Open a pull request with the proposed addition
- Explain why the example is helpful

### Typos and Grammar

- Pull requests for small fixes are welcome
- No issue required for obvious corrections

## What This Repository Is Not

This is methodology documentation, not the FYOS product itself. We cannot accept:

- Feature requests for the FYOS platform
- Bug reports for the FYOS application
- Requests for proprietary implementation details
- Production code contributions

For FYOS platform feedback, please use the contact form at [fyos.pro/contact](https://fyos.pro/contact).

## Process

### Issues

1. Check existing issues for duplicates
2. Use a descriptive title
3. Provide context and specific references

### Pull Requests

1. Fork the repository
2. Create a branch for your changes
3. Make focused, atomic commits
4. Write clear commit messages
5. Submit PR with description of changes

## Style Guidelines

### Documentation

- Use clear, direct language
- Avoid jargon without explanation
- Include examples where helpful
- Prefer formulas over prose for calculations

### Markdown

- One sentence per line (for readable diffs)
- Use fenced code blocks with language hints
- Tables for structured comparisons
- Headers for navigation
